# MTA News Radar 内容加工层 Spec

> 把之前留桩的 LLM 落地为"内容加工":对每主题 top-N 条目产出中文要点(核心总结 + 翻译合一),引擎侧生成、写进 JSON,前端只读。

## 背景

引擎产出多主题 JSON,但条目是原文——长摘要、大量英文。前端能展示却"难消化"。引擎里 LLM 还是桩(`LlmScorer` 委托 keyword)。本模块把 LLM 真正落地为内容加工:对每主题 top-N 条目产出中文要点,写进 `data/<topic>/latest.json` 供前端只读。

已就绪条目字段:`id / title / url / source_name / tier / summary / audio_url / published / sub_label / score`。

## 目标

- **G1**:对每主题排序后 top-N 条目,用 Claude(Haiku 4.5)产出 `title_zh`(中文标题)+ `summary_zh`(2-3 句中文核心摘要)。英文即翻译,中文即提炼。
- **G2**:成本可控——限 top-N、按条目 id 缓存(加工过不重复花钱)、数量/预算闸、密钥走环境变量。
- **G3**:加工在引擎侧(本地命令行 + 将来 Actions),结果写进 `data/<topic>/latest.json`;前端只读零成本。
- **G4**:优雅降级——无密钥/调用失败时,引擎照常产出(缺 zh 字段),前端回退原文,不崩不阻塞。

## 功能需求

- **F1 加工 top-N**:对每主题排序后前 N 条(N 可配,默认 24)产出 `title_zh` + `summary_zh`;N 之后不加工。
- **F2 要点内容**:`title_zh` = 中文标题(英文翻译 / 中文提炼);`summary_zh` = 2-3 句中文,讲清"是什么 + 为何值得看",基于条目原文 `title`+`summary`,忠实不编造。
- **F3 缓存**:按条目 `id` 缓存加工结果并持久化跨运行;已缓存的直接复用,不再调用 LLM。
- **F4 批量 + 预算闸**:多条目批量调用控成本;单次运行设数量上限,超限即停止本次新加工(已加工/已缓存照常写出)。
- **F5 写入 JSON**:结果写进对应条目(新增 `title_zh`/`summary_zh`);未加工条目两字段为空。
- **F6 优雅降级**:无 API key → 跳过加工、照常产出、不报错;单条目失败 → 该条无 zh,不中断其余条目与主题。
- **F7 密钥与配置**:API key 走环境变量(本地 / Actions Secret);`top-N`、模型、单次上限可配。
- **F8 成本可观测**:产出元信息记录「加工条数 / 命中缓存数 / 调用 LLM 数」。

## 非功能需求

- **N1 成本**:单次受 top-N + 缓存 + 单次上限三重约束;稳态(多数已缓存)增量成本≈只加工新条目。
- **N2 性能**:加工是网络 IO 密集 → 批量 + 有限并发控总时长,不阻塞未加工主题产出。
- **N3 安全**:API key 不硬编码、不入库、不入日志。
- **N4 可测**:LLM 客户端可注入假实现(测试不走网络);缓存/选取/降级逻辑可单测。
- **N5 依赖反转**:加工层依赖一个"中文要点生成"接口,不把 Claude SDK 焊进 pipeline——换 provider/model 不动编排。
- **N6 规范**:snake_case / PascalCase / 类型注解。
- **N7 确定性**:同条目同输入,缓存命中即稳定输出,不因重跑漂移。

## 不做的事

- **LLM 关系打分**:本模块只做"加工",相关性打分仍走 keyword;LLM 打分留后续。
- **前端分板块呈现**:单独模块(消费本模块的 zh 字段)。
- **Actions 部署 / Pages**:单独模块。本模块本地带 key 跑通即可。
- **全量加工**:不碰 top-N 之外的条目。
- **多语言**:只产中文。
- **抓正文 / 长总结**:只基于已有 `title+summary` 产 2-3 句,不爬全文。
- **缓存的部署级持久化机制**(committed vs Actions cache)留 plan / 部署模块定;本模块只要求"缓存可持久化复用"。

## 验收标准

- **AC1**(F1/F2):对一主题跑加工 → 前 N 条得非空中文 `title_zh`+`summary_zh`,N 之后为空。
- **AC2**(F2):英文条目 `title_zh` 是中文译名;中文条目 `title_zh` 是中文;`summary_zh` 为 2-3 句中文。
- **AC3**(F3):同数据第二次跑 → 全命中缓存,`llm_calls=0`(元信息可见),zh 字段不变。
- **AC4**(F4):单次上限 K、无缓存且条目 > K → 本次只新加工 K 条,其余无 zh,不报错。
- **AC5**(F6):不设 key 跑全流程 → 无 zh 字段、退出 0、不崩;前端回退原文。
- **AC6**(F6):单条目模拟 LLM 抛错 → 该条无 zh、其余正常。
- **AC7**(F5/F8):`latest.json` 条目含 `title_zh/summary_zh`;元信息含 `processed/from_cache/llm_calls` 计数。
- **AC8**(N4/N5):加工层用注入的假"要点生成器"完成全部单测,不触达网络。
