# MTA News Radar 前端消费 zh 字段 Spec

> 范围:纯前端。让卡片消费内容加工层产出的 `title_zh`/`summary_zh`,无则回退原文。

## 背景

内容加工层已产出并验证 `title_zh`/`summary_zh`(DeepSeek 实测:英文条目 → 干净中文标题+2-3句摘要),写进 `data/<topic>/latest.json` 的 top-N 条目。现有前端(`index.html` + `assets/radar.js`,左侧栏 + 卡片 + 收听 + `cleanSummary`)只显示英文原标题 / 清洗后摘要,尚未消费 zh。本模块是当初从加工层划出的"前端消费 zh"独立模块。

## 目标

- 卡片优先显示中文要点;无 zh 时回退原文。
- 不破坏现有功能(收听 / 分数 / 源健康 / 分页 / 移动端)。

## 功能需求

- **F1 标题**:有 `title_zh` → 中文作主标题 + 英文原 `title` 作灰色小字副标题;无 → 仅原文标题(现状)。
- **F2 摘要**:有 `summary_zh` → 显示它;无 → `cleanSummary(summary)`(现状)。
- **F3 回退**:top-N 之外 / 未加工 / 降级条目自然回退原文,不空不错。
- **F4 链接**:中文主标题仍是指向原 `url` 的链接;原文副标题为纯文本。
- **F5 安全**:中文/原文均 `textContent` 渲染,防 XSS。

## 非功能需求

- **N1** 纯前端,不动引擎 / 数据。
- **N2** `textContent` 渲染,防注入。
- **N3** 不破坏现有:收听链接 / 分数徽章 / 时间 / 源健康 / 加载更多 / 移动端抽屉。

## 不做的事

- AI / 机器翻译 徽章标记。
- 摘要展开看全文。
- 中英双语切换开关。
- 引擎 / 数据结构改动。

## 验收标准

- **AC1**:带 `title_zh` 的条目 → 中文主标题 + 英文小字副标题;点中文主标题在新标签打开原 `url`。
- **AC2**:带 `summary_zh` → 摘要显示中文;无 zh 的条目 → 回退原文标题 + `cleanSummary`。
- **AC3**:top-N 之外(无 zh)条目正常显示原文,不空不错。
- **AC4**:中文与原文文本均无 HTML 注入(textContent)。
- **AC5**:收听 / 分数 / 时间 / 源健康 / 加载更多 / 移动端不受影响。
