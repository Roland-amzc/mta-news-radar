# MTA News Radar 前端消费 zh 字段 Checklist

> 起本地 server,用带真实 zh 的数据(DeepSeek 跑出的)在浏览器观察验证。覆盖 frontend-zh-spec.md AC1–AC5。

## 标题(AC1)
- [ ] 带 `title_zh` 的条目 → 卡片主标题为中文,其下一行灰色小字为英文原标题(验证:对照 latest.json 的 title_zh 与 title)。
- [ ] 点中文主标题 → 新标签打开原 `url`(验证:点击核对地址)。

## 摘要(AC2)
- [ ] 带 `summary_zh` 的条目 → 摘要显示中文(验证:对照 summary_zh)。
- [ ] 无 zh 的条目 → 回退原文标题 + cleanSummary 后的摘要(验证:找 top-N 之外的条目观察)。

## 回退(AC3)
- [ ] top-N 之外 / 未加工条目正常显示英文原标题,无空标题、无报错(验证:滚到列表靠后或切到未加工主题)。

## 安全(AC4)
- [ ] 中文与原文文本均无 HTML 残留 / 无脚本执行(验证:DevTools 看 DOM 为文本节点;Console 无错)。

## 不破坏现有(AC5)
- [ ] 收听链接 / 分数徽章 / 时间 / 源健康摘要+详情 / 加载更多 / 移动端抽屉 均正常(验证:逐一交互)。

## 端到端
- [ ] 场景:`quant_factor`(带 zh)→ 中文主标题 + 英文副标题 + 中文摘要;切到一个未加工/空 zh 的情形 → 干净回退原文(结果:首屏即中文、回退无异常)。
